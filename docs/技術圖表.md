## 2. 工作流程圖（Workflow）

雲端排程每輪的 9 個步驟，以及伺服器端的更新流程。

```mermaid
flowchart TD
    START(["排程觸發<br/>每 8h／16強起 4h"]) --> S1["1 讀 fixtures.json<br/>找未來 48h 內待預測比賽"]
    S1 --> CHK{已進 16 強?}
    CHK -->|是| RESCHED["把排程改為每 4h"]
    CHK -->|否| S2
    RESCHED --> S2["2 每場研究 ≥50 來源<br/>產生預測 + 繁中理由<br/>獨立寫入 predictions/match_N__runid.json"]
    S2 --> S3["3 搜尋新完成比賽<br/>寫入 results.json"]
    S3 --> S4["4 update_accuracy.py<br/>比對賽前最後預測 vs 結果"]
    S4 --> S5["5 sync_to_site.py<br/>data → site/data"]
    S5 --> S6["6 同步新預測到 Notion"]
    S6 --> S7["7 push_to_github.py + git push"]
    S7 --> S8["8 npm build + 重新發布 pplx.app"]
    S8 --> S9{有新預測/結果?}
    S9 -->|是| NOTIFY["9 發繁中 in-app 通知"]
    S9 -->|否| SILENT["靜默結束"]

    S7 -.push 事件.-> SRV1["伺服器：webhook/timer 偵測新 commit"]
    SRV1 --> SRV2["update.sh：git reset --hard"]
    SRV2 --> SRVCHK{只動到 site/data?}
    SRVCHK -->|是| SRV3["docker compose restart（快）"]
    SRVCHK -->|否, 動到程式碼| SRV4["docker compose up -d --build"]
```

## 3. 狀態圖（State Diagram）

單一比賽（match）在系統中的生命週期狀態。

```mermaid
stateDiagram-v2
    [*] --> 已排程: fixtures.json 載入賽程
    已排程 --> 待預測: 進入未來 48h 窗口
    待預測 --> 已預測: 研究 ≥50 來源並產生預測
    已預測 --> 已預測: 下一輪重新預測<br/>（新 run_id，舊批次保留）
    已預測 --> 進行中: 比賽開賽 (kickoff)
    進行中 --> 已完成: 終場，寫入 results.json
    已完成 --> 已評分: update_accuracy.py<br/>比對賽前最後預測
    已評分 --> [*]: 命中率計入 accuracy.json

    note right of 已預測
        每場可有多筆預測批次
        檔名含 run_id，永不覆蓋
    end note
    note right of 已評分
        outcome_correct（1X2）
        exact_score_correct（X:X）
    end note
```

## 4. 循序圖（Sequence Diagram）

一輪完整排程從觸發到使用者看到更新的端到端互動。

```mermaid
sequenceDiagram
    autonumber
    participant CR as 排程器
    participant ENG as 預測引擎（雲端）
    participant GH as GitHub
    participant NT as Notion
    participant PX as pplx.app
    participant SV as 你的伺服器
    participant U as 使用者

    CR->>ENG: 觸發（每 8h／4h）
    ENG->>ENG: 讀 fixtures，找待預測比賽
    loop 每場比賽
        ENG->>ENG: 研究 ≥50 來源
        ENG->>ENG: 產生 X:X + 理由 + 信心度
    end
    ENG->>ENG: 搜尋新結果 → results.json
    ENG->>ENG: update_accuracy.py 計算命中率
    ENG->>NT: 同步新預測頁面
    ENG->>GH: push data/（commit）
    ENG->>PX: npm build + 重新發布
    GH-->>SV: webhook（push 事件，HMAC 驗證）
    SV->>GH: git pull（reset --hard）
    SV->>SV: docker compose restart / up -d --build
    U->>SV: HTTPS 開啟 wc2026.toniclab.ai
    SV-->>U: 登入 → 賽程 / 預測卡 / 準確率儀表板
    U->>PX: （或）開啟雲端版
    PX-->>U: 同源資料展示
```

## 5. 實體關聯圖（ER Diagram）

資料模型以 JSON 檔案儲存（非關聯式資料庫），以下用 ER 圖表達邏輯實體與關聯。`MATCH`（場次號）是貫穿全系統的主鍵。

```mermaid
erDiagram
    TOURNAMENT ||--o{ GROUP : 包含
    TOURNAMENT ||--o{ FIXTURE : 安排
    GROUP ||--o{ TEAM : 含有
    TEAM ||--o{ FIXTURE : 對戰
    FIXTURE ||--o{ PREDICTION : 被預測
    FIXTURE ||--o| RESULT : 產生
    PREDICTION ||--o| EVALUATION : 被評分
    RESULT ||--o| EVALUATION : 比對
    PREDICTION ||--o| NOTION_PAGE : 同步為

    TOURNAMENT {
        string tournament "FIFA World Cup 2026"
        array hosts "USA, Canada, Mexico"
        int total_matches "104"
        int teams "48"
        datetime last_updated
    }
    GROUP {
        string group_id PK "A–L"
        array teams "4 隊"
    }
    TEAM {
        string name PK "國家隊名"
        string group_id FK
    }
    FIXTURE {
        int match PK "場次號 1–104"
        string stage "Group A / R16 / ..."
        string group FK
        date date
        string kickoff_local
        string home FK
        string away FK
        string venue
        string city

        string home
```

## 6. 資料字典（Data Dictionary）

### `fixtures.json` — 賽程 Master List

| 欄位 | 型別 | 說明 |
|------|------|------|
| `tournament` | string | 賽事名稱（FIFA World Cup 2026） |
| `hosts` | array\<string\> | 主辦國：USA、Canada、Mexico |
| `total_matches` | int | 總場次（104） |
| `teams` | int | 參賽隊數（48） |
| `groups` | object | 組別 → 隊伍清單（A–L，每組 4 隊） |
| `fixtures` | array\<object\> | 全部 104 場賽程，欄位見下 |
| `fixtures[].match` | int | **場次號（主鍵，1–104）** |
| `fixtures[].stage` | string | 階段（Group A、Round of 16…） |
| `fixtures[].group` | string | 組別代號（A–L；淘汰賽為空） |
| `fixtures[].date` | date | 比賽日期 `YYYY-MM-DD` |
| `fixtures[].kickoff_local` | string | 當地開賽時間 `HH:MM` |
| `fixtures[].home` / `away` | string | 主／客隊名 |
| `fixtures[].venue` / `city` | string | 球場 / 城市 |
| `fixtures[].status` | string | `NS`（未開賽）／`FT`（完場） |
| `fixtures[].result` | object\|null | 完場比分 `{home,away}`，未完場為空 |

### `predictions/match_{N}__{run_id}.json` — 單場單批次預測（永不覆蓋）

| 欄位 | 型別 | 說明 |
|------|------|------|
| `match` | int | 場次號（對應 fixtures） |
| `run_id` | string | **預測批次（主鍵，`YYYY-MM-DDTHHMMZ`）** |
| `run_timestamp` | datetime | 該輪 UTC 觸發時間 |
| `stage` / `home` / `away` | string | 階段、主隊、客隊 |
| `kickoff` / `venue` | string | 開賽時間、球場 |
| `model` | string | 預測模型標記（`research-pplx`） |
| `prediction.score` | object | `{home,away}` 預測比分（整數） |
| `prediction.scoreline` | string | 比分字串（如 `0:2`） |
| `prediction.outcome` | enum | `home` / `draw` / `away` |
| `prediction.win_prob` | object | `{home,draw,away}` 機率（合計≈1） |
| `prediction.confidence` | float | 信心度 0–1 |
| `reasoning.summary` | string | 繁中分析摘要 |
| `reasoning.key_factors` | array\<string\> | 關鍵因素（球員狀態/傷停/戰術/場地/輿論…） |
| `reasoning.consensus_lean` | string | 主流共識傾向 |
| `reasoning.dissent` | string | 反向／少數意見 |

### `results.json` — 已完成比賽結果

| 欄位 | 型別 | 說明 |
|------|------|------|
| `last_updated` | datetime | 最後更新時間（UTC） |
| `results[].match` | int | 場次號 |
| `results[].stage` | string | 階段 |
| `results[].home` / `away` | string | 主／客隊 |
| `results[].score` | object | `{home,away}` 最終比分 |
| `results[].outcome` | enum | `home` / `draw` / `away` |
| `results[].recorded_at` | datetime | 記錄時間 |
| `results[].source` | string | 結果來源 URL |

### `accuracy.json` — 滾動準確率

| 欄位 | 型別 | 說明 |
|------|------|------|
| `metrics.total_evaluated` | int | 已評分比賽數 |
| `metrics.outcome_correct` | int | 勝負（1X2）命中數 |
| `metrics.outcome_accuracy` | float | 勝負命中率（0–1） |
| `metrics.exact_score_correct` | int | 比分（X:X）命中數 |
| `metrics.exact_score_accuracy` | float | 比分命中率（0–1） |
| `by_stage` | object | 各階段分項統計 |
| `evaluations[]` | array | 每場評分明細（match / run_id / 命中與否） |

### Notion DB「世界盃 2026 AI 預測追蹤」屬性

| 屬性 | Notion 型別 | 說明 |
|------|-------------|------|
| 比賽 | title | 頁面標題 |
| 場次 | number | 場次號 |
| 階段 | select | 小組賽/32強/16強/八強/四強/季軍戰/決賽 |
| 主隊 / 客隊 | text | 隊名 |
| AI預測比分 | text | 如 `0:2` |
| 預測結果 | select | 主勝 / 和局 / 客勝 |
| 信心度 | number (percent) | 0–1 |
| 來源數 | number | 蒐集來源數量 |
| 最終比分 | text | 完場後回填 |
| 勝負命中 / 比分命中 | checkbox | 評分後回填 |
| 預測批次 | text | run_id |
| 分析摘要 | text | ≤1900 字 |
| 更新時間 | date (datetime) | ISO 時間 |

> 同步邏輯：每輪預測為每場建立新頁面；比賽完成後，結果同步會更新該場「賽前最後一筆預測」頁面的最終比分與命中欄位。
