# 世界盃 2026 AI 預測中心 — 自架部署說明

一個全端網站（Express + Vite + React + Tailwind），含密碼登入、賽程 Master List、AI 預測卡（附 50+ 來源連結）、結果與準確率儀表板。所有介面皆為繁體中文。

---

## 一、最快上手（Docker，建議）

需求：已安裝 Docker。

```bash
# 1. 解壓縮後進入資料夾
cd wc2026-bundle

# 2. 建置映像
docker build -t wc2026-ai .

# 3. 啟動（請把密碼換成你自己的）
docker run -d --name wc2026-ai -p 5000:5000 \
  -e SITE_PASSWORD='你的新密碼' \
  wc2026-ai
```

開啟瀏覽器進入 `http://伺服器IP:5000`，輸入密碼登入即可。

### 使用 docker-compose（更方便）

```bash
cd wc2026-bundle
# 本站預設無密碼、公開。若要自架時加密碼，先取消 docker-compose.yml 內 SITE_PASSWORD 註解並填入後再執行：
docker compose up -d
```

> 注意：在 docker-compose.yml 中，YAML 會把 `$$` 解析成一個 `$`。
> 若你設定的密碼含有 `$` 字元，請在 compose 檔內自行加倍跳脫；或直接用 `docker run -e` 方式設定最單純。

## 二、不使用 Docker（直接用 Node.js 跑）

需求：Node.js 20 以上。

```bash
cd wc2026-bundle/site
npm ci                 # 安裝相依套件
npm run build          # 建置（產出 dist/public 與 dist/index.cjs）

# 啟動正式伺服器
SITE_PASSWORD='你的新密碼' PORT=5000 node dist/index.cjs
```

搭配 pm2 常駐（可選）：

```bash
npm install -g pm2
cd wc2026-bundle/site
SITE_PASSWORD='你的新密碼' pm2 start dist/index.cjs --name wc2026-ai
pm2 save
```

## 五、專案結構

```
wc2026-bundle/
├── Dockerfile              # Docker 映像定義
├── docker-compose.yml      # 一鍵啟動設定
├── .dockerignore
├── 部署說明.md             # 本文件
├── CRON_RUNBOOK.md         # 自動更新排程的完整執行手冊
├── notion_config.json      # Notion 同步設定（資料庫 ID、欄位對應）
├── update_accuracy.py      # 重新計算準確率（比對預測 vs 實際結果）
├── sync_to_site.py         # 把 data/ 同步到 site/data/
├── build_notion_pages.py   # 由預測 JSON 產生 Notion 頁面屬性
├── build_fixtures.py       # 產生 fixtures.json（賽程 Master List）
└── site/                   # 網站本體
    ├── client/             # React 前端（登入頁、儀表板、預測卡）
    ├── server/             # Express 後端（API 路由、資料讀取）
    ├── shared/             # 共用型別
    ├── data/               # ★ 所有資料來源（見下方）
    │   ├── fixtures.json       # 104 場賽程 Master List
    │   ├── results.json        # 已完成比賽的最終比分
    │   ├── accuracy.json       # 準確率統計
    │   └── predictions/        # 每次執行獨立保存的預測 JSON
    └── package.json
```

## 六、資料模型（data/ 資料夾）

這是整個系統的核心。網站只是把這些 JSON 渲染出來。

- **fixtures.json** — 賽程 Master List，104 場小組賽 + 淘汰賽路徑，48 支球隊分 A–L 組。
- **results.json** — 已完成比賽的最終比分（`results[]`）。
- **predictions/match_{場次}__{run_id}.json** — 每場比賽、每個執行批次「獨立一份」的預測檔，永不覆蓋舊批次。
  - `run_id` 格式為觸發時的 UTC 時間戳 `YYYY-MM-DDTHHMMZ`。
  - 內容含：預測比分、勝負機率、信心度、繁中分析理由（摘要／關鍵因素／共識傾向／反方觀點），以及 50 處以上的來源連結。
  - 結構範本見 `site/data/predictions/SCHEMA.md`。
- **accuracy.json** — 由 `update_accuracy.py` 自動計算，比對「開賽前最後一筆預測」與實際結果，得出勝負命中率與比分命中率。

## 七、如何更新預測與結果（自動化流程）

每一輪要做的事，完整步驟寫在 `CRON_RUNBOOK.md`，摘要如下：

1. 找出未來 48 小時內、尚無結果的即將開賽比賽。
2. 對每場比賽蒐集最新情報（傷兵、球員狀態、評論員、官方、論壇、社群、KOL、YouTube、博彩、模型），每場 ≥ 50 處來源，產生預測與繁中理由，寫入 `data/predictions/match_{N}__{run_id}.json`（獨立批次，不覆蓋）。
3. 把新完成比賽的最終比分寫入 `data/results.json`。
4. 執行 `python3 update_accuracy.py` 重算準確率。
5. 執行 `python3 sync_to_site.py` 把資料同步進網站。
6.（可選）用 `build_notion_pages.py` + `notion_config.json` 同步到 Notion。
7. 重新建置並重啟容器讓新資料生效：
   ```bash
   docker compose build && docker compose up -d
   # 或不用 docker：cd site && npm run build 後重啟 node 程序
   ```

> 建議排程：每日一次（HKT 13:00）；淘汰賽（32 強起）同樣維持每日一次。
> 若你用 docker-compose 的 volume 掛載 `./site/data`，更新資料後通常重啟容器即可，不必每次重建映像。

## 八、API 端點（供你整合）

所有 GET 端點需在 header 帶 `x-site-auth: <你的密碼>`。

| 方法 | 路徑 | 說明 |
|------|------|------|
| POST | `/api/login` | 帶 `{ password }`，回傳 `{ ok, token }` |
| GET | `/api/fixtures` | 104 場賽程 |
| GET | `/api/results` | 已完成比賽結果 |
| GET | `/api/predictions` | 各場次的預測（依場次分組） |
| GET | `/api/accuracy` | 準確率統計 |
| GET | `/api/status` | 系統狀態摘要 |

---

## 九、注意事項

- 登入採用記憶體 token（不使用 localStorage），重新整理頁面會需要再次登入，這是刻意設計。
- 正式上線請務必改掉預設密碼。
- 資料更新後需重新建置前端或重啟伺服器，網站才會顯示最新內容。
