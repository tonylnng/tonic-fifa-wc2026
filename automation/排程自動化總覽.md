# 世界盃 2026 — 排程自動化總覽

這份文件說明「每輪自動更新」的完整邏輯,以及 repo 內每個自動化檔案的用途,方便你日後參考或自行接手。

整套系統分三層：

```
雲端排程（預測引擎，每8小時 / 16強起每4小時）
        │  產生預測 → 更新結果 → 算準確率 → 同步 Notion
        │  → push data/ 到 GitHub → 重新發布 pplx.app → 發通知
        ▼
GitHub repo（tonylnng/tonic-fifa-wc2026，資料中轉站）
        │  你的伺服器偵測到新 commit
        ▼
你的伺服器（展示端，git pull + 重啟容器，見 deploy/）
```

預測運算全部在雲端排程跑,你的伺服器只負責讀取與展示,零運算負擔、不需任何 API 金鑰。

## 每輪執行的 9 個步驟

完整可執行版見 repo 根目錄 `CRON_RUNBOOK.md`。摘要：

| 步驟 | 動作 | 對應檔案 |
|------|------|----------|
| 1 | 讀 `data/fixtures.json`,找出未來 48 小時內、尚無結果的即將開賽比賽。若進 16 強則把排程改為每 4 小時 | `data/fixtures.json` |
| 2 | 每場研究最新情報（傷兵/狀態/評論員/官方/論壇/社群/KOL/YouTube/博彩/模型）,≥50 來源,產生預測 + 繁中理由,獨立寫入 `data/predictions/match_{N}__{run_id}.json`（不覆蓋舊批次） | `data/predictions/SCHEMA.md` |
| 3 | 搜尋新完成比賽,最終比分寫入 `data/results.json` | `data/results.json` |
| 4 | 重算準確率（比對開賽前最後一筆預測 vs 實際結果） | `update_accuracy.py` |
| 5 | 把 `data/` 同步進 `site/data/` | `sync_to_site.py` |
| 6 | 把新預測同步到 Notion 資料庫 | `build_notion_pages.py` + `notion_config.json` |
| 7 | 把更新後的 `data/` commit + push 到 GitHub | `push_to_github.py` |
| 8 | 重新建置並重新發布 pplx.app 網站（沿用 site_id） | `site/`（npm run build） |
| 9 | 有新預測/結果則發繁中通知,否則靜默結束 | — |

## 檔案清單

### 資料（`data/` 與 `site/data/`）
- `fixtures.json` — 104 場賽程 Master List（48 隊分 A–L 組 + 淘汰賽路徑）
- `results.json` — 已完成比賽最終比分
- `accuracy.json` — 勝負命中率、比分命中率統計
- `predictions/match_{場次}__{run_id}.json` — 每場每批次獨立保存的預測（永不覆蓋）
- `predictions/SCHEMA.md` — 預測 JSON 結構範本

### 自動化腳本（repo 根目錄）
- `update_accuracy.py` — 重算 `accuracy.json`。直接執行：`python3 update_accuracy.py`
- `sync_to_site.py` — 複製 `data/` → `site/data/`。直接執行：`python3 sync_to_site.py`
- `build_notion_pages.py` — 由預測 JSON 產生 Notion 頁面屬性
- `push_to_github.py` — 把最新 `data/` 複製進 repo 並 commit（路徑可用環境變數 `WC_SRC`/`WC_REPO`/`WC_DST` 覆寫）。push 由呼叫端帶 git 憑證執行
- `build_fixtures.py` — 重新產生 `fixtures.json`（建立賽程用,平時不需動）
- `notion_config.json` — Notion 資料庫 ID 與欄位對應
- `CRON_RUNBOOK.md` — 排程每輪的完整執行手冊

### 伺服器自動部署（`deploy/`）
見 `deploy/自動部署說明.md`：webhook 即時觸發（方案 A）、systemd timer 輪詢（方案 B）、`update.sh` 核心更新腳本。

## 排程設定（在 Perplexity 上執行）

目前由 Perplexity 的排程系統執行,設定如下：

- **排程 ID**：`c53acfd7`
- **頻率**：每 8 小時（UTC 0/8/16 = HKT 08:00/16:00/00:00）；進入 16 強後改每 4 小時（`0 0,4,8,12,16,20 * * *`）
- **執行內容**：依 `CRON_RUNBOOK.md` 跑上述 9 步
- **管理**：在這個對話的 [排程頁面](https://www.perplexity.ai/computer/tasks/829d0b9f-7fd8-425f-9e6e-9b6e9435bc52) 可暫停/修改/刪除

> 注意：步驟 2 的「研究 ≥50 來源並產生預測」與步驟 6/8 的「Notion 同步、pplx.app 重新發布」依賴 Perplexity 的研究與發布能力。若你日後要把預測引擎完全搬到自己環境,需自備來源蒐集與 LLM 推理管線；本 repo 的腳本（步驟 4/5/7）與資料結構可直接沿用。

## 想自己手動跑一輪（在有資料的環境）

```bash
python3 update_accuracy.py     # 重算準確率
python3 sync_to_site.py        # 同步進網站
python3 push_to_github.py      # 複製 data/ 進 repo 並 commit
git push origin master         # 推送（需 git 憑證）
cd site && npm run build        # 重新建置網站
```
